浅谈一下 World Model：一个被叫烂了、但其实定义很清楚的词（锐评版）

作者：dung defender  
来源：知乎专栏 · 大白话讲AI （不推公式版）  
时间：2026年4月1日

背景：World Model 原本是个很朴素的东西

标准定义：
World Model 的核心是描述一个动力学系统。其数学表达非常清晰：
s_{t+1} = f(s_t, a_t)

输入：当前系统状态 s_t 和当前动作 a_t。
输出：下一个状态 s_{t+1}。

这里的“动力学系统”是一个通用概念，不仅限于物理系统，任何能写成“当前状态 + 当前动作 rightarrow 下一状态”的系统都适用。

一旦有了这个动力学模型 f，决策问题就转化为标准的最优控制问题：

begin{aligned}
max_{a_0,dots,a_{T-1}} quad & sum_{t=0}^{T-1} r(s_t,a_t) + r_T(s_T) \
text{s.t.} quad & s_{t+1} = f(s_t,a_t), quad t=0,dots,T-1, \
& s_0 text{ given.}
end{aligned}

核心价值：
这个框架不依赖特定先验，非常通用。例如：
机器人：S是机器人状态，A是电机驱动，R是任务奖励。
围棋：S是棋局，A是落子，R是胜负。
语言：S是上下文，A是下一个Token，R是文本质量。

锐评 1：Sutton 的 bitter lesson 从未说只有纯数据驱动才可扩展。Closed-form solution、Optimization/Search、Data-driven 三者相辅相成。关键在于避免引入不必要的 Bias，而不在于“站队”。

为什么 World Model 这件事很重要

拥有精确的动力学模型 f 后，我们可以在模型上直接进行强化学习、规划和控制优化，而无需在真实环境中试错。

痛点：
大量真实世界的 f 根本无法通过传统还原论（还原论：把事物分解为部分来理解）的方式写出来。例如：
布料模拟（精度与速度难兼顾）。
刚体、软体、流体耦合系统（计算极重）。
大量 Agent 的复杂交互（规则写不全）。

如果能用神经网络学到一个近似的 f，就能解决这些原本无法处理的复杂系统问题。

锐评 2：World Model 的核心是建模复杂的动力系统，关键字是“动”。那些不涉及真实动态、仅在静态场景挪动的东西，包装成 World Model 有些牵强。

祖师爷：给 Schmidhuber 一点 Credit

必须承认 Jürgen Schmidhuber（LSTM 之父）的贡献。他在 2018 年的论文《World Models》是早期用神经网络建模环境动力学并结合规划的代表性工作。尽管欧洲学派声量减弱，但其开创性不可磨灭。

问题难点：一个可用的 World Model 难在哪

一个可用的 World Model 需要解决准确性（Accuracy）和一致性（Consistency）。本文重点讨论准确性，这至少涉及三个视角的设计：

4.1 系统表征 S：你到底想让模型“看见”什么

这是决定问题难度的关键。主流有三种路线：

a. 显式 3D 表征 (Explicit 3D)
    定义：NeRF、Mesh、GS 等。
    优点：结构清晰，紧凑，适合接优化和模拟器。
    缺点：
        带有强烈的还原论假设，难以表达软组织、衣物褶皱等复杂细节。
        数据成本极高，难以获取海量高质量的 3D 数据。

b. 2D 表征 (2D Video/Pixels)
    定义：直接使用视频像素。
    优点：数据极多（互联网视频海），无需转码。
    缺点：
        物理正确性难判断：看着像对了，不代表物理上对。
        计算量大且冗余：模型被迫为背景像素的明暗变化等无关任务的熵买单，而非关注动力学本质。

c. 隐式表征 (Implicit/JEPA)
    定义：提取抽象的、压缩后的内部表征。
    优点：不背诵像素，而是理解关键状态，方向最合理。
    缺点：
        面对多峰分布（Multi-modal）时，预测“平均答案”往往在语义上是错的。
        现有的压缩往往是“任务无关”的统一压缩，而人类认知是任务相关的（做不同事关注不同信息）。

小结：C 路线方向最合理，但坑最多。

4.2 模型：你到底打算拿什么去拟合 f

RNN (Recurrent Neural Networks)
    理论：NC1 递归计算模型，天然适合有状态递推的动力系统。
    实践：难以并行，长序列优化困难，工业界不讨喜。
    评价：理论上更对，实践上像“受苦”。

Transformer
    理论：TC0 有限层前馈并行结构，计算复杂度受限。
    实践：并行友好，吃大数据能力强，工程生态好。
    评价：对于低复杂度动力学够用，但对于高复杂度、多主体的系统可能力不从心。且在视觉/控制领域，如何实现类似文本模型的“推理时多算一点”（CoT）尚不明确。

核心问题：我们是否对目标 f 的复杂度有靠谱估计？如果没有，架构讨论往往是玄学。

4.3 训练方式：你到底要用什么 Loss 去逼近 f

Regression (回归)
    优点：简单稳定，适合高精度数值拟合。
    缺点：天生怕多峰。面对多结果场景，往往学到的是“所有可能性的平均尸体”。

Diffusion (扩散模型)
    优点：表达力强，能建模多峰分布，生成质量高。
    缺点：
        计算代价大。
        优化目标是分布匹配，而非显式的点对点动力学监督。
        不一定能保证状态误差小、物理量不漂移、接触关系正确。

总结

World Model 不是一个单点技术，而是一个系统工程。它牵涉到表征、计算、控制、规划、训练目标和数据形式的共同设计。

当前的局面是：一个词火了，先经历“定义膨胀”，再经历“概念去泡沫化”。希望 World Model 最终能回归到真正有价值的问题上。
